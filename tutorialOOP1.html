OOP 1

What is an object?

An object is something that has an identity, a state, and a behaviour

Identity - identifiers
State - variables
Behavior - methods

What is a class?

A class is a blueprint for a type of object. Multiple objects can be created using the same class.

You can think of classes and objects as cookies and cookie cutters. The class is the cookie cutter, and the objects are the cookies. You use the cookie cutter to create many cookies with the same shape, but the cookies may have different flavors. There is only one cookie cutter, but you can make as many cookies as youâ€™d like.

Static methods

Static methods are methods that belong to a class definition. They are a part of the class but they are not a part of the objects of the class, so they can be called without creating an object.

What is a constructor?

A constructor is used to create the objects of a class. It initializes objects and sets initial values for object attributes. It is called when objects are created. If you do not create a constructor, java creates a default constructor. You can also create multiple constructors with different parameters. This can be helpful when you have many objects that will have the same attribute for an object.

Encapsulation

Variables and methods can be public or private. Making things private means they can only be accessed from within the class. Accessing private variables can be done using set and get methods. Get methods get the variable value, while set methods change the variable value
